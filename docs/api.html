Public API for the PING Platform, backed by Convex.

How to call Convex functions

All queries and mutations are called as JSON-RPC style HTTP requests:

Function type Method Path
Query POST {CONVEX_URL}/api/query
Mutation POST {CONVEX_URL}/api/mutation

Request body shape

{
  "path": "<module>:<function>",
  "args": { ... },
  "format": "json"
}

Authentication

Pass a Bearer JWT token issued by WorkOS in the Authorization header:

Authorization: Bearer <token>

Webhooks

Inbound webhook endpoints (server-to-server, no user auth)

WorkOS webhook receiver

Receives events from WorkOS (user lifecycle, directory sync, etc.). Currently handles: user.created, user.updated, dsync.user.created, dsync.user.deleted.

Request Body schema: application/json
required
event
required
string

WorkOS event type

data
required
object

Event payload (shape depends on event type)

Responses

Convex deployment URL

/{convexUrl}/webhooks/workos

Request samples

Content type

{

    • "first_name": "Alice",
    • "last_name": "Smith",
    • "profile_picture_url": "https://cdn.example.com/alice.jpg"
    }
}
